Key Concepts

  1. Protocol: A set of rules for transferring data, for example https://.
  2. Uniform Resource Locator (URL): A web address that is a unique identifier, digital addresses that direct browsers to a specific website.
  3. Methods: Common methods that include GET, POST, PUT, DELETE, give a request purpose.
  4. HTTPS (TLS): A secure version of HTTP, where TLS is the protocol that encrypts data for secure communication over a network.
  5. 404 Not Found: The requested resource could not be found.
  6. Caching: A temporary storage for regularly accessed data, imporves performance by allowing future requests to be served faster
